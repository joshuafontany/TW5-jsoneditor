<p>JSON Editor takes a JSON Schema and uses it to generate an HTML form.<br>
It has full support for JSON Schema version 3 and 4 and can integrate with several popular CSS frameworks (bootstrap, foundation, and jQueryUI).</p>
<p>Check out an interactive demo (demo.html): <a href="http://rawgithub.com/json-editor/json-editor/master/docs/demo.html" rel="nofollow">http://rawgithub.com/json-editor/json-editor/master/docs/demo.html</a></p>

<h3><a id="user-content-optional-requirements" class="anchor" aria-hidden="true" href="#optional-requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Optional Requirements</h3>

<span class="pl-c"><span class="pl-c">//</span> `getEditor` will return null if the path is invalid</span>

  <span class="pl-smi">name</span>.<span class="pl-en">setValue</span>(<span class="pl-s"><span class="pl-pds">"</span>John Smith<span class="pl-pds">"</span></span>);

<h3><a id="user-content-validate" class="anchor" aria-hidden="true" href="#validate"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Validate</h3>
<p>When feasible, JSON Editor won't let users enter invalid data.  This is done by
using input masks and intelligently enabling/disabling controls.</p>
<p>However, in some cases it is still possible to enter data that doesn't validate against the schema.</p>
<p>You can use the <code>validate</code> method to check if the data is valid or not.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-c"><span class="pl-c">//</span> Validate the editor's current value against the schema</span>
<span class="pl-k">var</span> errors <span class="pl-k">=</span> <span class="pl-smi">editor</span>.<span class="pl-en">validate</span>();

<span class="pl-k">if</span>(<span class="pl-smi">errors</span>.<span class="pl-c1">length</span>) {
  <span class="pl-c"><span class="pl-c">//</span> errors is an array of objects, each with a `path`, `property`, and `message` parameter</span>
  <span class="pl-c"><span class="pl-c">//</span> `property` is the schema keyword that triggered the validation error (e.g. "minLength")</span>
  <span class="pl-c"><span class="pl-c">//</span> `path` is a dot separated path into the JSON object (e.g. "root.path.to.field")</span>
  <span class="pl-en">console</span>.<span class="pl-c1">log</span>(errors);
}
<span class="pl-k">else</span> {
  <span class="pl-c"><span class="pl-c">//</span> It's valid!</span>
}</pre></div>
<p>By default, this will do the validation with the editor's current value.
If you want to use a different value, you can pass it in as a parameter.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-c"><span class="pl-c">//</span> Validate an arbitrary value against the editor's schema</span>
<span class="pl-k">var</span> errors <span class="pl-k">=</span> <span class="pl-smi">editor</span>.<span class="pl-en">validate</span>({
  value<span class="pl-k">:</span> {
    to<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>test<span class="pl-pds">"</span></span>
  }
});</pre></div>
<h3><a id="user-content-listen-for-changes" class="anchor" aria-hidden="true" href="#listen-for-changes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Listen for Changes</h3>
<p>The <code>change</code> event is fired whenever the editor's value changes.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">editor</span>.<span class="pl-en">on</span>(<span class="pl-s"><span class="pl-pds">'</span>change<span class="pl-pds">'</span></span>,<span class="pl-k">function</span>() {
  <span class="pl-c"><span class="pl-c">//</span> Do something</span>
});

<span class="pl-smi">editor</span>.<span class="pl-en">off</span>(<span class="pl-s"><span class="pl-pds">'</span>change<span class="pl-pds">'</span></span>,function_reference);</pre></div>
<p>You can also watch a specific field for changes:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">editor</span>.<span class="pl-c1">watch</span>(<span class="pl-s"><span class="pl-pds">'</span>path.to.field<span class="pl-pds">'</span></span>,<span class="pl-k">function</span>() {
  <span class="pl-c"><span class="pl-c">//</span> Do something</span>
});

<span class="pl-smi">editor</span>.<span class="pl-c1">unwatch</span>(<span class="pl-s"><span class="pl-pds">'</span>path.to.field<span class="pl-pds">'</span></span>,function_reference);</pre></div>
<p>Or watch all fields (Similar to the "onchange" event, but tracks the field changed)</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> <span class="pl-en">watcherCallback</span> <span class="pl-k">=</span> <span class="pl-k">function</span>(<span class="pl-smi">path</span>) {
  <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>field with path: [<span class="pl-pds">"</span></span> <span class="pl-k">+</span> path <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">"</span>] changed to [<span class="pl-pds">"</span></span> <span class="pl-k">+</span> <span class="pl-c1">JSON</span>.<span class="pl-c1">stringify</span>(<span class="pl-c1">this</span>.<span class="pl-en">getEditor</span>(path).<span class="pl-en">getValue</span>()) <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">"</span>]<span class="pl-pds">"</span></span>);
  <span class="pl-c"><span class="pl-c">//</span> Do something</span>
}
<span class="pl-k">for</span> (<span class="pl-k">var</span> key <span class="pl-k">in</span> <span class="pl-smi">editor</span>.<span class="pl-smi">editors</span>) {
  <span class="pl-k">if</span> (<span class="pl-smi">editor</span>.<span class="pl-smi">editors</span>.<span class="pl-en">hasOwnProperty</span>(key) <span class="pl-k">&amp;&amp;</span> key <span class="pl-k">!==</span> <span class="pl-s"><span class="pl-pds">'</span>root<span class="pl-pds">'</span></span>) {
    <span class="pl-smi">editor</span>.<span class="pl-c1">watch</span>(key, <span class="pl-smi">watcherCallback</span>.<span class="pl-en">bind</span>(editor, key));
  }
}</pre></div>

<h2><a id="user-content-json-schema-support" class="anchor" aria-hidden="true" href="#json-schema-support"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>JSON Schema Support</h2>
<p>JSON Editor fully supports version 3 and 4 of the JSON Schema <a href="http://json-schema.org/latest/json-schema-core.html" rel="nofollow">core</a> and <a href="http://json-schema.org/latest/json-schema-validation.html" rel="nofollow">validation</a> specifications.<br>
Some of The <a href="http://json-schema.org/latest/json-schema-hypermedia.html" rel="nofollow">hyper-schema</a> specification is supported as well.</p>
<h3><a id="user-content-ref-and-definitions" class="anchor" aria-hidden="true" href="#ref-and-definitions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>$ref and definitions</h3>
<p>JSON Editor supports schema references to external URLs and local definitions.  Here's an example showing both:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Full Name<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>$ref<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>#/definitions/name<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>location<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>$ref<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>http://mydomain.com/geo.json<span class="pl-pds">"</span></span>
    }
  },
  <span class="pl-s"><span class="pl-pds">"</span>definitions<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>minLength<span class="pl-pds">"</span></span>: <span class="pl-c1">5</span>
    }
  }
}</pre></div>
<p>Local references must point to the <code>definitions</code> object of the root node of the schema.
So, <code>#/customkey/name</code> will throw an exception.</p>
<p>If loading an external url via Ajax, the url must either be on the same domain or return the correct HTTP cross domain headers.
If your URLs don't meet this requirement, you can pass in the references to JSON Editor during initialization (see Usage section above).</p>
<p>Self-referential $refs are supported.  Check out <code>examples/recursive.html</code> for usage examples.</p>
<h3><a id="user-content-hyper-schema-links" class="anchor" aria-hidden="true" href="#hyper-schema-links"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>hyper-schema links</h3>
<p>The <code>links</code> keyword from the hyper-schema specification can be used to add links to related documents.</p>
<p>JSON Editor will use the <code>mediaType</code> property of the links to determine how best to display them.<br>
Image, audio, and video links will display the media inline as well as providing a text link.</p>
<p>Here are a couple examples:</p>
<p>Simple text link</p>
<pre lang="js+jinja"><code>{
  "title": "Blog Post Id",
  "type": "integer",
  "links": [
    {
      "rel": "comments",
      "href": "/posts/{{self}}/comments/",
      // Optional - set CSS classes for the link
      "class": "comment-link open-in-modal primary-text"
    }
  ]
}
</code></pre>
<p>Make link download when clicked</p>
<pre lang="js+jinja"><code>{
  "title": "Document filename",
  "type": "string",
  "links": [
    {
      "rel": "Download File",
      "href": "/documents/{{self}}",
      // Can also set `download` to a string as per the HTML5 spec
      "download": true
    }
  ]
}
</code></pre>
<p>Show a video preview (using HTML5 video)</p>
<pre lang="js+jinja"><code>{
  "title": "Video filename",
  "type": "string",
  "links": [
    {
      "href": "/videos/{{self}}.mp4",
      "mediaType": "video/mp4"
    }
  ]
}
</code></pre>
<p>The <code>href</code> property is a template that gets re-evaluated every time the value changes.
The variable <code>self</code> is always available.  Look at the <strong>Dependencies</strong> section below for how to include other fields or use a custom template engine.</p>
<h3><a id="user-content-property-ordering" class="anchor" aria-hidden="true" href="#property-ordering"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Property Ordering</h3>
<p>There is no way to specify property ordering in JSON Schema (although this may change in v5 of the spec).</p>
<p>JSON Editor introduces a new keyword <code>propertyOrder</code> for this purpose.  The default property order if unspecified is 1000.  Properties with the same order will use normal JSON key ordering.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>prop1<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>prop2<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>propertyOrder<span class="pl-pds">"</span></span>: <span class="pl-c1">10</span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>prop3<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>propertyOrder<span class="pl-pds">"</span></span>: <span class="pl-c1">1001</span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>prop4<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>propertyOrder<span class="pl-pds">"</span></span>: <span class="pl-c1">1</span>
    }
  }
}</pre></div>
<p>In the above example schema, <code>prop1</code> does not have an order specified, so it will default to 1000.
So, the final order of properties in the form (and in returned JSON data) will be:</p>
<ol>
<li>prop4 (order 1)</li>
<li>prop2 (order 10)</li>
<li>prop1 (order 1000)</li>
<li>prop3 (order 1001)</li>
</ol>
<h3><a id="user-content-default-properties" class="anchor" aria-hidden="true" href="#default-properties"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Default Properties</h3>
<p>The default behavior of JSON Editor is to include all object properties defined with the <code>properties</code> keyword.</p>
<p>To override this behaviour, you can use the keyword <code>defaultProperties</code> to set which ones are included:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {<span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>},
    <span class="pl-s"><span class="pl-pds">"</span>age<span class="pl-pds">"</span></span>: {<span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>integer<span class="pl-pds">"</span></span>}
  },
  <span class="pl-s"><span class="pl-pds">"</span>defaultProperties<span class="pl-pds">"</span></span>: [<span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>]
}</pre></div>
<p>Now, only the <code>name</code> property above will be included by default.  You can use the "Object Properties" button
to add the "age" property back in.</p>
<h3><a id="user-content-format" class="anchor" aria-hidden="true" href="#format"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>format</h3>
<p>JSON Editor supports many different formats for schemas of type <code>string</code>.  They will work with schemas of type <code>integer</code> and <code>number</code> as well, but some formats may produce weird results.
If the <code>enum</code> property is specified, <code>format</code> will be ignored.</p>
<p>JSON Editor uses HTML5 input types, so some of these may render as basic text input in older browsers:</p>
<ul>
<li>color</li>
<li>date</li>
<li>datetime</li>
<li>datetime-local</li>
<li>email</li>
<li>month</li>
<li>password</li>
<li>number</li>
<li>range</li>
<li>tel</li>
<li>text</li>
<li>textarea</li>
<li>time</li>
<li>url</li>
<li>week</li>
</ul>
<p>Here is an example that will show a color picker in browsers that support it:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {

      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>color<span class="pl-pds">"</span></span>
    }
  }
}</pre></div>
<h4><a id="user-content-string-editors-input-attributes" class="anchor" aria-hidden="true" href="#string-editors-input-attributes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>String Editors Input Attributes</h4>
<p>You can set custom attributes such as <strong>placeholder</strong>, <strong>class</strong> and <strong>data-</strong> on the input field using the special options keyword <code>inputAttributes</code>.</p>
<p>Like this:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Full Name<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>options<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>inputAttributes<span class="pl-pds">"</span></span>: {
          <span class="pl-s"><span class="pl-pds">"</span>placeholder<span class="pl-pds">"</span></span>:  <span class="pl-s"><span class="pl-pds">"</span>your name here...<span class="pl-pds">"</span></span>,
          <span class="pl-s"><span class="pl-pds">"</span>class<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>myclass<span class="pl-pds">"</span></span>
        }
      }
    }
  }
}</pre></div>

  <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>checkbox<span class="pl-pds">"</span></span>
}</pre></div>
<h4><a id="user-content-arrays" class="anchor" aria-hidden="true" href="#arrays"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Arrays</h4>
<p>The default array editor takes up a lot of screen real estate.  The <code>table</code> and <code>tabs</code> formats provide more compact UIs for editing arrays.</p>
<p>The <code>table</code> format works great when every array element has the same schema and is not too complex.</p>
<p>The <code>tabs</code> format can handle any array, but only shows one array element at a time. It has tabs on the left for switching between items.</p>
<p>The <code>tabs-top</code> format place tabs on the top.</p>
<p>Here's an example of the <code>table</code> format:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>table<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
      }
    }
  }
}</pre></div>
<p>For arrays of enumerated strings, you can also use the <code>select</code> or <code>checkbox</code> format.  These formats require a very specific schema to work:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>uniqueItems<span class="pl-pds">"</span></span>: <span class="pl-c1">true</span>,
  <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>enum<span class="pl-pds">"</span></span>: [<span class="pl-s"><span class="pl-pds">"</span>value1<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>value2<span class="pl-pds">"</span></span>]
  }
}</pre></div>
<p>By default, the <code>checkbox</code> editor (multiple checkboxes) will be used if there are fewer than 8 enum options.  Otherwise, the <code>select</code> editor (a multiselect box) will be used.</p>
<p>You can override this default by passing in a format:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>select<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>uniqueItems<span class="pl-pds">"</span></span>: <span class="pl-c1">true</span>,
  <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>enum<span class="pl-pds">"</span></span>: [<span class="pl-s"><span class="pl-pds">"</span>value1<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>value2<span class="pl-pds">"</span></span>]
  }
}</pre></div>
<h4><a id="user-content-objects" class="anchor" aria-hidden="true" href="#objects"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Objects</h4>
<p>The default object layout is one child editor per row.  The <code>grid</code> format will instead put multiple child editors per row.
This can make the editor much more compact, but at a cost of not guaranteeing child editor order.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: { <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span> }
  },
  <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>grid<span class="pl-pds">"</span></span>
}</pre></div>
<p>The <code>categories</code> format groups properties in top-tabbed panels, one for each object or array property plus one that groups all added or other types of properties.<br>
Panel tabs titles came from object or array titles and for the grouping panel it defaults to "Basic", unless  <code>basicCategoryTitle</code> is defined.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: { <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span> }
  },
  <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>categories<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>basicCategoryTitle<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Main<span class="pl-pds">"</span></span>
}</pre></div>
<p>Demo page will look like this:</p>
<p><a target="_blank" rel="noopener noreferrer" href="/json-editor/json-editor/blob/master/docs/images/categoriesDemo.png"><img src="/json-editor/json-editor/raw/master/docs/images/categoriesDemo.png" alt="Categories format" style="max-width:100%;"></a></p>
<h2><a id="user-content-editor-options" class="anchor" aria-hidden="true" href="#editor-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Editor Options</h2>
<p>Editors can accept options which alter the behavior in some way.</p>
<ul>
<li><code>collapsed</code> - If set to true, the editor will start collapsed (works for objects and arrays)</li>
<li><code>disable_array_add</code> - If set to true, the "add row" button will be hidden (works for arrays)</li>
<li><code>disable_array_delete</code> - If set to true, all of the "delete" buttons will be hidden (works for arrays)</li>
<li><code>disable_array_delete_all_rows</code> - If set to true, just the "delete all rows" button will be hidden (works for arrays)</li>
<li><code>disable_array_delete_last_row</code> - If set to true, just the "delete last row" buttons will be hidden (works for arrays)</li>
<li><code>disable_array_reorder</code> - If set to true, the "move up/down" buttons will be hidden (works for arrays)</li>
<li><code>disable_collapse</code> - If set to true, the collapse button will be hidden (works for objects and arrays)</li>
<li><code>disable_edit_json</code> - If set to true, the Edit JSON button will be hidden (works for objects)</li>
<li><code>disable_properties</code> - If set to true, the Edit Properties button will be hidden (works for objects)</li>
<li><code>array_controls_top</code> - If set to true, array controls (add, delete etc) will be displayed at top of list (works for arrays)</li>
<li><code>enum_titles</code> - An array of display values to use for select box options in the same order as defined with the <code>enum</code> keyword. Works with schema using enum values.</li>
<li><code>expand_height</code> - If set to true, the input will auto expand/contract to fit the content.  Works best with textareas.</li>
<li><code>grid_columns</code> - Explicitly set the number of grid columns (1-12) for the editor if it's within an object using a grid layout.</li>
<li><code>hidden</code> - If set to true, the editor will not appear in the UI (works for all types)</li>
<li><code>input_height</code> - Explicitly set the height of the input element. Should be a valid CSS width string (e.g. "100px").  Works best with textareas.</li>
<li><code>input_width</code> - Explicitly set the width of the input element. Should be a valid CSS width string (e.g. "100px").  Works for string, number, and integer data types.</li>
<li><code>remove_empty_properties</code> - If set to true for an object, empty object properties (i.e. those with falsy values) will not be returned by getValue().</li>
</ul>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>options<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>collapsed<span class="pl-pds">"</span></span>: <span class="pl-c1">true</span>
  },
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
    }
  }
}</pre></div>
<p>You can globally set the default options too if you want:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-smi">editors</span>.<span class="pl-c1">object</span>.<span class="pl-c1">options</span>.<span class="pl-smi">collapsed</span> <span class="pl-k">=</span> <span class="pl-c1">true</span>;</pre></div>
<h2><a id="user-content-dependencies" class="anchor" aria-hidden="true" href="#dependencies"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Dependencies</h2>
<p>Sometimes, it's necessary to have one field's value depend on another's.</p>
<p>The <code>dependencies</code> keyword from the JSON Schema specification is not nearly flexible enough to handle most use cases,
so JSON Editor introduces a couple custom keywords that help in this regard.</p>
<p>The first step is to have a field "watch" other fields for changes.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>first_name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>last_name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>full_name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>watch<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>fname<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>first_name<span class="pl-pds">"</span></span>,
        <span class="pl-s"><span class="pl-pds">"</span>lname<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>last_name<span class="pl-pds">"</span></span>
      }
    }
  }
}</pre></div>
<p>The keyword <code>watch</code> tells JSON Editor which fields to watch for changes.</p>
<p>The keys (<code>fname</code> and <code>lname</code> in this example) are alphanumeric aliases for the fields.</p>
<p>The values (<code>first_name</code> and <code>last_name</code>) are paths to the fields.  To access nested properties of objects, use a dot for separation (e.g. "path.to.field").</p>
<p>By default paths are from the root of the schema, but you can make the paths relative to any ancestor node with a schema <code>id</code> defined as well.  This is especially useful within arrays.  Here's an example:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>id<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>arr_item<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>first_name<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
      },
      <span class="pl-s"><span class="pl-pds">"</span>last_name<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
      },
      <span class="pl-s"><span class="pl-pds">"</span>full_name<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
        <span class="pl-s"><span class="pl-pds">"</span>watch<span class="pl-pds">"</span></span>: {
          <span class="pl-s"><span class="pl-pds">"</span>fname<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>arr_item.first_name<span class="pl-pds">"</span></span>,
          <span class="pl-s"><span class="pl-pds">"</span>lname<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>arr_item.last_name<span class="pl-pds">"</span></span>
        }
      }
    }
  }
}</pre></div>
<p>Now, the <code>full_name</code> field in each array element will watch the <code>first_name</code> and <code>last_name</code> fields within the same array element.</p>
<h3><a id="user-content-templates" class="anchor" aria-hidden="true" href="#templates"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Templates</h3>
<p>Watching fields by itself doesn't do anything.  For the example above, you need to tell JSON Editor that <code>full_name</code> should be <code>fname [space] lname</code>.
JSON Editor uses a javascript template engine to accomplish this.  A barebones template engine is included by default (simple <code>{{variable}}</code> replacement only), but many of the most popular template engines are also supported:</p>
<ul>
<li>ejs</li>
<li>handlebars</li>
<li>hogan</li>
<li>markup</li>
<li>mustache</li>
<li>swig</li>
<li>underscore</li>
</ul>
<p>You can change the default by setting <code>JSONEditor.defaults.options.template</code> to one of the supported template engines:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-c1">options</span>.<span class="pl-smi">template</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>handlebars<span class="pl-pds">'</span></span>;</pre></div>
<p>You can set the template engine on a per-instance basis as well:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> editor <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">JSONEditor</span>(element,{
  schema<span class="pl-k">:</span> schema,
  template<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>hogan<span class="pl-pds">'</span></span>
});</pre></div>
<p>Here is the completed <code>full_name</code> example using the default barebones template engine:</p>
<pre lang="js+jinja"><code>{
  "type": "object",
  "properties": {
    "first_name": {
      "type": "string"
    },
    "last_name": {
      "type": "string"
    },
    "full_name": {
      "type": "string",
      "template": "{{fname}} {{lname}}",
      "watch": {
        "fname": "first_name",
        "lname": "last_name"
      }
    }
  }
}
</code></pre>
<h3><a id="user-content-enum-values" class="anchor" aria-hidden="true" href="#enum-values"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Enum Values</h3>
<p>Another common dependency is a drop down menu whose possible values depend on other fields.  Here's an example:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>possible_colors<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
      }
    },
    <span class="pl-s"><span class="pl-pds">"</span>primary_color<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>
    }
  }
}</pre></div>
<p>Let's say you want to force <code>primary_color</code> to be one of colors in the <code>possible_colors</code> array.  First, we must tell the <code>primary_color</code> field to watch the <code>possible_colors</code> array.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>primary_color<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>watch<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>colors<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>possible_colors<span class="pl-pds">"</span></span>
    }
  }
}</pre></div>
<p>Then, we use the special keyword <code>enumSource</code> to tell JSON Editor that we want to use this field to populate a drop down.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>primary_color<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>watch<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>colors<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>possible_colors<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>enumSource<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>colors<span class="pl-pds">"</span></span>
  }
}</pre></div>
<p>Now, anytime the <code>possible_colors</code> array changes, the dropdown's values will be changed as well.</p>
<p>This is the most basic usage of <code>enumSource</code>.  The more verbose form of this property supports
filtering, pulling from multiple sources, constant values, etc..
Here's a more complex example (this uses the Swig template engine syntax to show some advanced features)</p>
<pre lang="js+jinja"><code>{
  // An array of sources
  "enumSource": [
    // Constant values
    ["none"],
    {
      // A watched field source
      "source": "colors",
      // Use a subset of the array
      "slice": [2,5],
      // Filter items with a template (if this renders to an empty string, it won't be included)
      "filter": "{% if item !== 'black' %}1{% endif %}",
      // Specify the display text for the enum option
      "title": "{{item|upper}}",
      // Specify the value property for the enum option
      "value": "{{item|trim}}"
    },
    // Another constant value at the end of the list
    ["transparent"]
  ]
}
</code></pre>
<p>You can also specify a list of static items with a slightly different syntax:</p>
<pre lang="js+jinja"><code>{
  "enumSource": [{
      // A watched field source
      "source": [
        {
          "value": 1,
          "title": "One"
        },
        {
          "value": 2,
          "title": "Two"
        }
      ],
      "title": "{{item.title}}",
      "value": "{{item.value}}"
    }]
  ]
}
</code></pre>
<p>The colors examples used an array of strings directly.  Using the verbose form, you can
also make it work with an array of objects.  Here's an example:</p>
<pre lang="js+jinja"><code>{
  "type": "object",
  "properties": {
    "possible_colors": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "text": {
            "type": "string"
          }
        }
      }
    },
    "primary_color": {
      "type": "string",
      "watch": {
        "colors": "possible_colors"
      },
      "enumSource": [{
        "source": "colors",
        "value": "{{item.text}}"
      }]
    }
  }
}
</code></pre>
<p>All of the optional templates in the verbose form have the properties <code>item</code> and <code>i</code> passed into them. <code>item</code> refers to the array element.  <code>i</code> is the zero-based index.</p>
<h3><a id="user-content-dynamic-headers" class="anchor" aria-hidden="true" href="#dynamic-headers"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Dynamic Headers</h3>
<p>The <code>title</code> keyword of a schema is used to add user friendly headers to the editing UI.  Sometimes though, dynamic headers, which change based on other fields, are helpful.</p>
<p>Consider the example of an array of children.  Without dynamic headers, the UI for the array elements would show <code>Child 1</code>, <code>Child 2</code>, etc..<br>
It would be much nicer if the headers could be dynamic and incorporate information about the children, such as <code>1 - John (age 9)</code>, <code>2 - Sarah (age 11)</code>.</p>
<p>To accomplish this, use the <code>headerTemplate</code> property.  All of the watched variables are passed into this template, along with the static title <code>title</code> (e.g. "Child"), the 0-based index <code>i0</code> (e.g. "0" and "1"), the 1-based index <code>i1</code>, and the field's value <code>self</code> (e.g. <code>{"name": "John", "age": 9}</code>).</p>
<pre lang="js+jinja"><code>{
  "type": "array",
  "title": "Children",
  "items": {
    "type": "object",
    "title": "Child",
    "headerTemplate": "{{ i1 }} - {{ self.name }} (age {{ self.age }})",
    "properties": {
      "name": { "type": "string" },
      "age": { "type": "integer" }
    }
  }
}
</code></pre>
<h3><a id="user-content-custom-template-engines" class="anchor" aria-hidden="true" href="#custom-template-engines"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Custom Template Engines</h3>
<p>If one of the included template engines isn't sufficient,
you can use any custom template engine with a <code>compile</code> method.  For example:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> myengine <span class="pl-k">=</span> {
  <span class="pl-en">compile</span><span class="pl-k">:</span> <span class="pl-k">function</span>(<span class="pl-smi">template</span>) {
    <span class="pl-c"><span class="pl-c">//</span> Compile should return a render function</span>
    <span class="pl-k">return</span> <span class="pl-k">function</span>(<span class="pl-smi">vars</span>) {
      <span class="pl-c"><span class="pl-c">//</span> A real template engine would render the template here</span>
      <span class="pl-k">var</span> result <span class="pl-k">=</span> template;
      <span class="pl-k">return</span> result;
    }
  }
};

<span class="pl-c"><span class="pl-c">//</span> Set globally</span>
<span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-c1">options</span>.<span class="pl-smi">template</span> <span class="pl-k">=</span> myengine;

<span class="pl-c"><span class="pl-c">//</span> Set on a per-instance basis</span>
<span class="pl-k">var</span> editor <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">JSONEditor</span>(element,{
  schema<span class="pl-k">:</span> schema,
  template<span class="pl-k">:</span> myengine
});</pre></div>
<h2><a id="user-content-language-and-string-customization" class="anchor" aria-hidden="true" href="#language-and-string-customization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Language and String Customization</h2>
<p>JSON Editor uses a translate function to generate strings in the UI.  A default <code>en</code> language mapping is provided.</p>
<p>You can easily override individual translations in the default language or create your own language mapping entirely.</p>
<pre lang="js+jinja"><code>// Override a specific translation
JSONEditor.defaults.languages.en.error_minLength =
  "This better be at least {{0}} characters long or else!";


// Create your own language mapping
// Any keys not defined here will fall back to the "en" language
JSONEditor.defaults.languages.es = {
  error_notset: "propiedad debe existir"
};
</code></pre>
<p>By default, all instances of JSON Editor will use the <code>en</code> language.  To override this default, set the <code>JSONEditor.defaults.language</code> property.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-c1">language</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>es<span class="pl-pds">"</span></span>;</pre></div>
<h2><a id="user-content-button-custimization" class="anchor" aria-hidden="true" href="#button-custimization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Button Custimization</h2>
<p>All buttons have classnames in the format <code>json-editor-btntype-*</code>. Using these classnames you can choose if the button should have icon or label hidden. The icon is wrapped in an <code>I</code> tag, the label is wrapped in a <code>SPAN</code> tag.</p>
<p>Examples:</p>
<p>Hide the icon on "Object Properties" button:</p>
<div class="highlight highlight-source-css"><pre><span class="pl-e">.json-editor-btntype-properties</span> <span class="pl-ent">i</span> {
  <span class="pl-c1"><span class="pl-c1">display</span></span>: <span class="pl-c1">none</span>;
}</pre></div>
<p>Hide the text on "Object Properties" button:</p>
<div class="highlight highlight-source-css"><pre><span class="pl-e">.json-editor-btntype-properties</span> <span class="pl-ent">span</span> {
  <span class="pl-c1"><span class="pl-c1">display</span></span>: <span class="pl-c1">none</span>;
}</pre></div>
<h2><a id="user-content-custom-editor-interfaces" class="anchor" aria-hidden="true" href="#custom-editor-interfaces"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Custom Editor Interfaces</h2>
<p>JSON Editor contains editor interfaces for each of the primitive JSON types as well as a few other specialized ones.</p>
<p>You can add custom editors interfaces fairly easily.  Look at any of the existing ones for an example.</p>
<p>JSON Editor uses resolver functions to determine which editor interface to use for a particular schema or subschema.</p>
<p>Let's say you make a custom <code>location</code> editor for editing geo data.  You can add a resolver function to use this custom editor when appropriate. For example:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-c"><span class="pl-c">//</span> Add a resolver function to the beginning of the resolver list</span>
<span class="pl-c"><span class="pl-c">//</span> This will make it run before any other ones</span>
<span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-smi">resolvers</span>.<span class="pl-c1">unshift</span>(<span class="pl-k">function</span>(<span class="pl-smi">schema</span>) {
  <span class="pl-k">if</span>(<span class="pl-smi">schema</span>.<span class="pl-c1">type</span> <span class="pl-k">===</span> <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span> <span class="pl-k">&amp;&amp;</span> <span class="pl-smi">schema</span>.<span class="pl-smi">format</span> <span class="pl-k">===</span> <span class="pl-s"><span class="pl-pds">"</span>location<span class="pl-pds">"</span></span>) {
    <span class="pl-k">return</span> <span class="pl-s"><span class="pl-pds">"</span>location<span class="pl-pds">"</span></span>;
  }

  <span class="pl-c"><span class="pl-c">//</span> If no valid editor is returned, the next resolver function will be used</span>
});</pre></div>
<p>The following schema will now use this custom editor for each of the array elements instead of the default <code>object</code> editor.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>array<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>items<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>object<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>format<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>location<span class="pl-pds">"</span></span>,
    <span class="pl-s"><span class="pl-pds">"</span>properties<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>longitude<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>number<span class="pl-pds">"</span></span>
      },
      <span class="pl-s"><span class="pl-pds">"</span>latitude<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>number<span class="pl-pds">"</span></span>
      }
    }
  }
}</pre></div>
<p>If you create a custom editor interface that you think could be helpful to others, submit a pull request!</p>
<p>The possibilities are endless.  Some ideas:</p>
<ul>
<li>A compact way to edit objects</li>
<li>Radio button version of the <code>select</code> editor</li>
<li>Autosuggest for strings (like enum, but not restricted to those values)</li>
<li>Better editor for arrays of strings (tag editor)</li>
<li>Canvas based image editor that produces Base64 data URLs</li>
</ul>
<h2><a id="user-content-select2--selectize-support" class="anchor" aria-hidden="true" href="#select2--selectize-support"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Select2 &amp; Selectize Support</h2>

<h2><a id="user-content-custom-validation" class="anchor" aria-hidden="true" href="#custom-validation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Custom Validation</h2>
<p>JSON Editor provides a hook into the validation engine for adding your own custom validation.</p>
<p>Let's say you want to force all schemas with <code>format</code> set to <code>date</code> to match the pattern <code>YYYY-MM-DD</code>.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-c"><span class="pl-c">//</span> Custom validators must return an array of errors or an empty array if valid</span>
<span class="pl-smi">JSONEditor</span>.<span class="pl-smi">defaults</span>.<span class="pl-smi">custom_validators</span>.<span class="pl-c1">push</span>(<span class="pl-k">function</span>(<span class="pl-smi">schema</span>, <span class="pl-smi">value</span>, <span class="pl-smi">path</span>) {
